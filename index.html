Frequently Asked Question
What is the difference between var, let, and const
var: In javascript the var keyword is function-scoped, hoisted and initialized with undefined. It allows re-declaration.
let: The let keyword is block-scoped, hoisted but not initialized. Accessing the let variables before declaration result in ReferenceError. It doesn't allow re-declaration within the same scope.
const: const keyword is also block-scoped, hoisted but not initialized. Once assigned, const cannot be reassigned; however, if it holds an object or array, the properties can still be modified.
What is the difference between map(), forEach(), and filter()
forEach: The forEach method executes a provided function once for each array element. It is typically used to perform side effects rather than to transform the array.
map: The map method creates a new array populated with the results of calling a provided function on every element in the calling array. It is used to transform data.
filter: The filter method creates a new array with all elements that pass the test implemented by the provided function. It is used to include only certain elements from the original array.
Explain arrow functions, and how they are different from regular function
Arrow functions: Arrow functions are introduced in ES6. it is more concise way to define functions.They are best suited for short, simple functions, especially when you want to maintain a shorter, more readable syntax.
Difference:

Arrow functions and regular functions in JavaScript have key differences in syntax, behavior, and usage. Arrow functions (=>) provide a shorter and more concise syntax compared to regular functions, making them ideal for simple operations and callbacks. A major difference is how they handle this.

How javascript promise works?
In javascript promise is an asynchronous action that may complete at some point in the future and produce a value. This value is not necessarily known at the time of its creation. Once the value is produced, it notifies the user. Promises provide a robust way to wrap the result of asynchronous work, overcoming the problem of deeply nested callbacks.
How closure works in javascript?
In JavaScript, a closure is a fundamental concept that allows a function to remember and access variables from its surrounding scope, even after that outer function has finished executing.
When a function is created inside another function, the inner function act as a closure function. This closure includes the inner function itself and a reference to the outer function's variables.
